Next.js 14 中台主体 + RDS PostgreSQL + PM2 + Nginx/HTTPS + 舆情分析 Docker（同机）

适用分支：feature/gameops（发布前请合并至 main） ·  包管理：pnpm · 运行时：Node 20 LTS · 文档生成：2026-07-08

0方案总览

本项目是单进程长驻的一体化 Next.js 应用：instrumentation.ts 在进程内启动了 node-cron 调度器、日志清理、舆情报告 downloader，且 Agent 并发=1。因此它不能做 Serverless、不能多实例横向扩容 —— 只能由 PM2 以单实例（fork 模式）常驻。

组件选型说明
计算阿里云 ECS · Ubuntu 22.04 LTS建议 ≥ 2 vCPU / 4 GB（含 Docker 分析服务则 ≥ 4 vCPU / 8 GB）
数据库阿里云 RDS PostgreSQL 15/16内网连中台，参数 timezone=UTC
运行Node 20 + pnpm + PM2单实例 next start，开机自启
入口Nginx + HTTPS反代 3000；证书用 Let's Encrypt 或阿里云免费 SSL
舆情分析slg_analyzer Docker与中台同机，仅监听 127.0.0.1:8000
整体顺序：准备资源 → 建 RDS 库 → 装服务器环境 → 传代码 → 写 .env → 迁移+构建 → PM2 起服务 → Nginx 上 HTTPS → 起分析容器 → 验证清单。全程约 60–90 分钟。

1阿里云资源准备

  1. ECS 实例：Ubuntu 22.04 LTS，系统盘 ≥ 40 GB。若跑分析容器另挂一块数据盘或系统盘给到 80 GB+。记录公网 IP 与 VPC/交换机。
  2. RDS PostgreSQL：与 ECS 选同一 VPC、同一可用区（走内网、低延迟、免公网费）。
  3. 安全组（ECS）：入方向放行 80、443；22 仅对你的运维 IP 放行。不要对公网放行 3000／8000（只走本机回环）。
  4. 域名：把 A 记录解析到 ECS 公网 IP；国内服务器域名需完成 ICP 备案。
3000 与 8000 端口永不对公网开放。外部只经 Nginx（443）进入；分析服务只监听 127.0.0.1。

2RDS 数据库配置

  1. 创建数据库与账号：库名如 gameops，创建高权限账号（迁移需建表权限）。
  2. 白名单：把 ECS 的内网 IP 加入 RDS 白名单（或加整个 VPC 网段）。获取 RDS 内网连接地址。
  3. 时区：本项目要求数据库时区为 UTC。在 RDS 参数设置里把 timezone 设为 UTC 并应用（前端用 lib/format.ts 锁 Asia/Shanghai 渲染）。

组装连接串（稍后写进 .env 的 DATABASE_URL）：

postgresql://账号:密码@rds内网地址:5432/gameops?schema=public
密码若含 @ : / ? 等特殊字符，需做 URL 编码（例如 @→%40）。

3ECS 服务器初始化

SSH 登录 ECS 后依次执行。建议用普通用户 + sudo，不要长期用 root 跑应用。

4上传代码

推荐用 Git 拉取（后续更新一条 git pull 即可）。选一个部署目录，例如 /opt/gameops。

sudo mkdir -p /opt/gameops && sudo chown $USER:$USER /opt/gameops
cd /opt/gameops
git clone <你的仓库地址> .
git checkout main            # 发布分支

若仓库不便联网克隆，可本机打包上传（在 Windows PowerShell 里用 scp）：

# 本机：排除 node_modules/.next/.git 后打包
tar --exclude=node_modules --exclude=.next --exclude=.git -czf gameops.tgz .
scp gameops.tgz user@ECS公网IP:/opt/gameops/
# 服务器：解包
cd /opt/gameops && tar xzf gameops.tgz
.env、data/、storage/、public/uploads/ 已在 .gitignore 内 —— 它们不会随 Git 过来，属于服务器本地状态，下面单独处理。

5环境变量 .env

在项目根 /opt/gameops/.env 写入以下内容。先生成三个随机密钥：

# AUTH_SECRET（≥32 字符）
node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"
# ANALYSIS_SHARED_SECRET（与分析服务同源，hex）
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# OPINION_AES_KEY（32 字节 base64，加密 LLM apiKey 用）
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
# ── 数据库（RDS 内网地址）──
DATABASE_URL="postgresql://账号:密码@rds内网地址:5432/gameops?schema=public"

# ── 会话 JWT 密钥（上一步生成）──
AUTH_SECRET="<randomBytes(48) base64>"

# ── 应用对外地址：构建期注入，必须是最终 HTTPS 域名 ──
NEXT_PUBLIC_APP_URL="https://yourdomain.com"

# ── 飞书群机器人 webhook（阶段6，可先留空）──
FEISHU_WEBHOOK_URL=""

# ── 抖音 access_token（可选，可留空）──
DOUYIN_ACCESS_TOKEN=""

NODE_ENV="production"

# ── 阶段9 · 舆情监控 ──
ANALYSIS_BASE_URL="http://127.0.0.1:8000"
ANALYSIS_SHARED_SECRET="<randomBytes(32) hex>"
OPINION_AES_KEY="<randomBytes(32) base64>"
⚠ NEXT_PUBLIC_APP_URL 是构建期注入。凡 NEXT_PUBLIC_* 变量都会在 pnpm build 时被内联进前端包。必须在构建之前就设成最终域名；改了它必须重新 build，只重启不生效。
舆情三项 env 缺失只 warn 不阻塞其它功能（首次触发舆情接口时才懒失败）。若本次不上分析服务，可先留占位。AUTH_SECRET 一旦上线不要更换，否则所有登录态失效。
变量必填作用
DATABASE_URL是Prisma 连接 RDS
AUTH_SECRET是jose 签发/校验 JWT（httpOnly cookie）
NEXT_PUBLIC_APP_URL是构建期注入的对外地址
ANALYSIS_BASE_URL舆情分析服务地址（同机 127.0.0.1:8000）
ANALYSIS_SHARED_SECRET舆情与分析服务共享的 Bearer
OPINION_AES_KEY舆情AES-256-GCM 加密 LLM apiKey
FEISHU_WEBHOOK_URL / DOUYIN_ACCESS_TOKEN否飞书外发 / 抖音审核，可留空

6安装依赖 · 数据库迁移 · 构建

cd /opt/gameops

# 1) 安装依赖（会自动编译 argon2）
pnpm install --frozen-lockfile

# 2) 生成 Prisma Client
pnpm db:generate

# 3) 应用数据库迁移（生产用 deploy，不是 dev）
pnpm exec prisma migrate deploy

# 4)（首次上线可选）灌入种子数据 / 初始管理员账号
pnpm db:seed

# 5) 生产构建（此刻 .env 里的 NEXT_PUBLIC_APP_URL 会被内联）
pnpm build
用 prisma migrate deploy，不要用 db:migrate（=migrate dev）。dev 会尝试改写迁移历史、可能触发重置，属于开发命令，严禁在生产库执行。

确保运行时目录存在（Git 不带内容，只带 .gitkeep）：

mkdir -p data/raw data/logs storage/opinion-reports public/uploads
pnpm db:seed 是否含默认管理员账号，请对照 prisma/seed.ts；上线后立即修改任何默认口令。

7PM2 启动常驻

在项目根建 ecosystem.config.js：

module.exports = {
  apps: [{
    name: "gameops",
    script: "node_modules/next/dist/bin/next",
    args: "start -p 3000",
    cwd: "/opt/gameops",
    instances: 1,              // 必须单实例：进程内 cron/downloader
    exec_mode: "fork",         // 不可 cluster
    autorestart: true,
    max_memory_restart: "1G",
    env: { NODE_ENV: "production" }
  }]
};
严禁 instances: "max" 或 exec_mode: "cluster"。多实例会让 node-cron 调度器、舆情 downloader 重复触发，且违背 Agent 并发=1 语义。永远只跑一个实例。
cd /opt/gameops
pm2 start ecosystem.config.js

# 开机自启（按提示复制它输出的那条 sudo 命令执行一次）
pm2 startup
pm2 save

pm2 status
pm2 logs gameops --lines 50   # 看启动日志，确认 cron/downloader 已起

此时本机应可访问：curl -I http://127.0.0.1:3000 返回 200/3xx。

8Nginx 反向代理 + HTTPS

sudo apt -y install nginx
sudo tee /etc/nginx/sites-available/gameops > /dev/null <<'EOF'
server {
    listen 80;
    server_name yourdomain.com;

    client_max_body_size 50m;   # EXCEL/封面上传，放开体积

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 300s;   # 任务日志/长响应
    }
}
EOF
sudo ln -sf /etc/nginx/sites-available/gameops /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx

8.1 上 HTTPS

方式 A · Let's Encrypt（免费自动续期）：

sudo apt -y install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com
# certbot 会自动改写上面的 server 块，加 443 + 证书 + 80→443 跳转

方式 B · 阿里云免费 SSL：在阿里云申请证书，下载 Nginx 格式，把 listen 443 ssl; + ssl_certificate / ssl_certificate_key 指到证书文件，并加 80→443 跳转。

为什么必须 HTTPS：登录态是 httpOnly cookie，生产环境需 Secure 属性走 HTTPS 才安全；NEXT_PUBLIC_APP_URL 也应与此域名一致。

9舆情分析服务（slg_analyzer · 同机 Docker）

该服务是独立仓库 slg_analyzer（FastAPI 包装 run.py/run_public.py/run_combined.py），本项目通过中台代理调用。

  1. 把 slg_analyzer 传到服务器，例如 /opt/slg_analyzer。
  2. 在其 .env 里设与中台相同的 ANALYSIS_SHARED_SECRET（Bearer 同源），以及各 LLM 配置。
  3. 启动容器，仅绑定回环端口：
cd /opt/slg_analyzer
docker compose up -d
docker compose ps
# 从中台机验证（应带 Bearer 才通）
curl -H "Authorization: Bearer $ANALYSIS_SHARED_SECRET" http://127.0.0.1:8000/health
docker compose 的端口映射请写成 127.0.0.1:8000:8000，不要 0.0.0.0，避免分析服务暴露公网。中台侧 ANALYSIS_BASE_URL=http://127.0.0.1:8000 已匹配。
中台不落 OpinionReport 表：任务状态在分析服务的 SQLite，产物 HTML/JSON 由中台 downloader 每 10s 拉回 storage/opinion-reports/<taskId>/。所以这个目录必须持久化（见下一节）。具体见 docs/opinion-monitor.md。

10持久化目录与备份

路径内容随 Git备份策略
data/raw/爬虫原始 CSV 留底否随云盘快照
data/logs/Task 执行日志（90 天自动清理）否可不备份
storage/opinion-reports/舆情报告 HTML/JSON 产物否随云盘快照
public/uploads/活动封面 / 创作灵感上传否随云盘快照
RDS 全库业务数据单一事实源—RDS 自动备份
  • 数据库：在 RDS 控制台开启自动备份 + 保留策略，这是最重要的一环。
  • 本地磁盘：给 ECS 系统盘/数据盘配置自动快照策略，覆盖上面几个目录。
  • 确保这些目录的属主是运行 PM2 的用户，避免写入 EACCES。

11上线验证清单

  • https://yourdomain.com/login 能打开，证书有效（绿锁）。
  • 用管理员账号登录成功，按 role 正确跳转（ADMIN/OPERATOR → /operator，CREATOR → /dashboard）。
  • pm2 status 显示 gameops online，重启计数为 0。
  • pm2 logs gameops 中可见 cron-scheduler / downloader 启动日志，无 DATABASE_URL / 连接报错。
  • 后台能建 Job / 触发 Task（若爬虫机 Agent 已就绪）。
  • 上传一张活动封面 / 一个 EXCEL 参数，无 413（体积）报错 → 验证 Nginx client_max_body_size。
  • 舆情：ADMIN 保存 LLM 设置成功（AES 加密），触发一次报告，10s 后能在列表看到并打开 HTML。
  • 重启一次 ECS，确认 PM2 与 Docker 容器都开机自启。

12更新与回滚

日常更新（无破坏性迁移）

cd /opt/gameops
git pull
pnpm install --frozen-lockfile
pnpm db:generate
pnpm exec prisma migrate deploy
pnpm build
pm2 reload gameops        # 平滑重启
若本次改了 NEXT_PUBLIC_APP_URL 或任何 NEXT_PUBLIC_*，pnpm build 已覆盖它们；只 reload 不 build 不生效。

回滚

cd /opt/gameops
git log --oneline -5           # 找上一个好版本
git checkout <good-commit>
pnpm install --frozen-lockfile && pnpm db:generate && pnpm build
pm2 reload gameops
数据库迁移不会随代码回滚。若新版本含破坏性迁移，回滚代码前先评估数据兼容性；重大迁移上线前务必先在 RDS 做一次手动备份/快照。

13常见问题排查

现象排查方向
启动即崩 / DB 连接超时RDS 白名单是否含 ECS 内网 IP；DATABASE_URL 用的是内网地址；密码特殊字符是否 URL 编码。
502 Bad GatewayPM2 是否 online（pm2 status）；curl 127.0.0.1:3000 是否通；Nginx proxy_pass 端口。
时间显示差 8 小时RDS 参数 timezone 是否为 UTC；前端渲染由 lib/format.ts 锁 Asia/Shanghai。
上传 413 Request Entity Too LargeNginx client_max_body_size 调大（示例 50m）。
argon2 报错 / 装不上确认 build-essential 已装；删 node_modules 后重 pnpm install。
cron/舆情任务重复触发PM2 是否被误配成多实例/cluster；本项目必须单实例 fork。
舆情报告一直不出现分析容器是否 up；两侧 ANALYSIS_SHARED_SECRET 是否一致；ANALYSIS_BASE_URL 是否 127.0.0.1:8000；storage/opinion-reports/ 是否可写。
改了 NEXT_PUBLIC_APP_URL 不生效该变量构建期内联，必须 pnpm build 后再 reload。
